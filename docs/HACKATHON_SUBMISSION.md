# OpenAI Build Week 2026 Submission Checklist

This document is an English submission-preparation checklist. It does not replace the official rules.

## Project Evidence Included in This Repository

- [x] Education-track project source code.
- [x] English README with setup, environment, testing, architecture, privacy, dependencies, assets, Codex collaboration, GPT-5.6 use, and session evidence.
- [x] Spanish and Japanese README translations linked from the English README.
- [x] Dated git history beginning on 2026-07-17 and detailed Codex session records.
- [x] Primary Codex `/feedback` session ID: `019f6fca-6bee-7f33-9836-ae5f6f5fa65b`.
- [x] MIT license for a public repository option.
- [x] Judge testing instructions.
- [x] English demo-video script under three minutes.
- [x] Third-party dependencies and project-generated logo disclosed in the README.

## Required Before Devpost Submission

- [ ] Choose the **Education** category.
- [ ] Deploy the current commit and add the working public URL to Devpost.
- [ ] Configure production environment variables, especially `CRON_SECRET`, and verify one real conversion on the deployed URL.
- [ ] Record a public YouTube video under three minutes with audio. It must clearly show the working product and explain how Codex and GPT-5.6 were used.
- [ ] Add the public YouTube URL to Devpost.
- [ ] Add an English project description explaining features, audience, impact, and technical implementation.
- [ ] Add the repository URL. Make it public with this license, or share a private repository with `testing@devpost.com` and `build-week-event@openai.com`.
- [ ] Enter the primary Codex `/feedback` session ID on Devpost.
- [ ] Confirm that the final video contains no unlicensed music, third-party trademarks, or copyrighted material.
- [ ] Confirm that all submitted descriptions, test instructions, and video narration or captions are in English.

## Recommended Final Evidence Capture

- Screenshot one English, one Spanish, and one Japanese completed result.
- Save the deployed URL, YouTube URL, final commit SHA, and submission timestamp in the Devpost draft.
- Retain the local Codex session records and dated git history until judging ends.
